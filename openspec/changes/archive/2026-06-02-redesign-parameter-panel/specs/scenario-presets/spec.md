## ADDED Requirements

### Requirement: 具名情境管理
系統 SHALL 允許將一組參數儲存為具名情境，並提供切換、另存、匯出與匯入，取代原本以單一檔案進行的參數匯入/匯出。

#### Scenario: 切換內建情境
- **WHEN** 使用者從情境選擇器選擇一個內建情境（理想節拍 / 瓶頸情境 / 客戶現況）
- **THEN** 該情境的參數載入面板欄位，使用者可進一步套用以驅動模擬

#### Scenario: 另存目前參數為情境
- **WHEN** 使用者調整參數後選擇「另存」並輸入名稱
- **THEN** 系統以該名稱儲存目前參數為新的情境，並出現在情境選擇器中

#### Scenario: 內建情境為唯讀範本
- **WHEN** 使用者修改了某內建情境的參數
- **THEN** 系統不覆寫內建範本，而是引導使用者另存為自訂情境

### Requirement: 情境持久化與相容遷移
系統 SHALL 將情境集合持久化於 localStorage，並在偵測到舊版單一參數格式時自動遷移而不遺失既有設定。

#### Scenario: 遷移舊版參數
- **WHEN** 系統載入時偵測到舊版 `sim-params`（無版本標記的單組參數）
- **THEN** 系統將其包裝為一個具名情境並保留原值，後續以新版情境集合格式儲存

#### Scenario: 遷移失敗安全回退
- **WHEN** localStorage 內容損毀或無法解析
- **THEN** 系統回退為內建預設情境且不拋出例外、不中斷載入

### Requirement: 情境匯出與匯入
系統 SHALL 支援以情境為單位的 JSON 匯出與匯入。

#### Scenario: 匯出情境
- **WHEN** 使用者選擇匯出
- **THEN** 系統下載一個包含情境名稱、製程線與參數的 JSON 檔

#### Scenario: 匯入情境
- **WHEN** 使用者匯入一個有效的情境 JSON 檔
- **THEN** 系統將其加入情境集合並可於選擇器中選用；若檔案格式錯誤則顯示 inline 錯誤訊息而非阻斷式對話框
