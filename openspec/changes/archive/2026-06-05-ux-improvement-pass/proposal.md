## Why

模擬器功能已相當完整，但從 UX/UI 工程角度仍有幾項影響「易用性、可讀性、與決策說服力」的結構性問題：

1. **核心訊號被埋起來**：即時 UPH、已完成、預估完成、瓶頸警示原本是這個「瓶頸 what-if 工具」最該被看到的資料，但目前整個塞在預設收合的統計圖表覆蓋層內，平常完全看不到。
2. **控制列過載**：Header 一列放了約 15 個同等權重的按鈕（播放/速度/立即結果/重置/切換線/視圖/看板/比較/視角/參數/主題/說明），主次不分，新手要逐一掃描。
3. **缺乏 RWD 與簡報情境**：整站只有 1 條 media query（`prefers-reduced-motion`），版面固定寬度；本工具常被投影給主管看產能方案，但小螢幕會擠爆、字級 10–13px 在大螢幕一群人看不清。
4. **決策動線藏在彈窗**：方案比較表（釘選/並排）與瓶頸建議（套用 +1 並重跑）是產品真正的價值，卻是次級 modal，使用者容易沒發現。
5. **視覺語言不一致 / 可讀性**：emoji 與線條圖示混用；流程圖圖例是左上 10px 小字；站別狀態仍只靠顏色（滿框綠/空框橘/加工藍），未做色盲友善（AGV 車頂已加形狀，站別未加）。
6. **關鍵時刻設計不足**：達標結果視窗是純文字，未呈現成「成績單」；尚無結果可匯出、尚未開始模擬等空狀態缺乏引導。

## What Changes

- **生命徵象條（live vitals）**：把即時 UPH / 已完成 / 預估完成 / 🔴瓶頸 從統計圖表拆出來，做成一條常駐、輕量、隨模擬即時更新的列；統計圖表回歸「想細看時才展開」的角色。
- **控制列分層**：將 Header 控制依使用頻率/性質分群——主要（播放・速度・立即結果）、分析（看板・比較・瓶頸）、工具（視角・參數・主題・說明），降低掃描成本。
- **RWD + 簡報模式**：加入合理的版面斷點避免窄視窗擠爆；新增「簡報 / 大字模式」——只留流程圖 + 超大的 完成/UPH/ETA/瓶頸，給會議室投影使用。
- **決策動線抬升**：提升方案比較與瓶頸建議的可發現性（達標後主動引導「釘選比較」、瓶頸出現時的行動入口）。
- **視覺一致性 + 可讀性**：統一圖示風格、放大/重排流程圖圖例、站別狀態加上形狀/標記（色盲友善）、檢視關鍵字級對比。
- **成績單式結果 + 空狀態**：把達標結果視窗重做成一眼到位的成績單（產能・瓶頸・ETA + 匯出/比較 CTA）；補上空狀態引導。

## Capabilities

### New Capabilities
- `live-vitals-bar`: 常駐顯示即時 UPH / 已完成 / 預估完成 / 瓶頸指示，獨立於統計圖表覆蓋層。
- `command-bar-hierarchy`: Header 控制列依主要/分析/工具分層分群與視覺權重。
- `presentation-mode`: 版面斷點（RWD）與「簡報/大字」全螢幕 glanceable 模式。
- `result-scorecard`: 達標/計算結果以成績單呈現，並補齊空狀態引導。

### Modified Capabilities
- `scenario-comparison-table`: 提升決策動線可發現性（達標引導釘選比較、與瓶頸建議的行動連動）。
- `display-refinements`: 視覺語言一致性、流程圖圖例可讀性、站別狀態形狀編碼（色盲友善）、關鍵字級/對比。

## Impact

- 單一檔案 `ui/index.html`（純前端、狀態全在記憶體；不新增外部相依）：
  - Header（`#hdr` 控制列 `:245`）重排分群；新增簡報模式切換。
  - 底部統計區（`#bot` / `#sp` `:286`）：將即時統計/瓶頸抽出為常駐 vitals 條；圖表覆蓋層維持。
  - `updateProgressAndStats()` / `updateUI()`：vitals 條與簡報模式的即時刷新。
  - 結果視窗（`#ov` `:389`、`showFastResult`/`checkDone`）重做版面。
  - 流程圖圖例（`abf-hdr2` 等）與站別渲染（`buildABFSVG`）的視覺調整。
  - 新增 CSS 斷點與簡報模式樣式（目前僅 1 條 media query）。
- **不改變模擬引擎的派工/時序/產出計算**；所有變更為呈現層與資訊架構。
- 桌面版（Tauri WebView2）需以實機驗證版面不破、簡報模式正常（沿用本專案既有的 CDP 驗證流程）。

## Non-Goals

- 不改動模擬數值邏輯、派工演算法、既有 UPH/瓶頸計算。
- 不引入 UI 框架或外部相依（維持單檔純前端）。
- 不重做自訂線精靈、不動 blueprint 模型。
- 不處理多語系 / i18n（維持現有中文）。
